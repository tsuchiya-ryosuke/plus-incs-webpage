# PLUS UI文言トーンガイド（生徒向け / 先生向け）

## 目的
- 同じ機能でも、受け手に合わせた語調で伝わり方を最適化する。
- 画面実装・レビュー・翻訳の際に、文体ぶれを減らす。

## 1. 対象別トーン

### 生徒向け（チャット / ガイド）
- 基本トーン: **やわらかい敬体**（です・ます）
- 目標: 安心感・伴走感・自己開示のしやすさ
- 言い回し:
  - 推奨: 「〜してみましょう」「〜でも大丈夫です」「一緒に考えましょう」
  - 非推奨: 「〜しなさい」「当然」「普通は」
- 語彙:
  - 推奨: 身近で短い語（例: 「相談する」「気持ちを整理する」）
  - 非推奨: 教務寄りの硬い語（例: 「指導」「査定」「規定に基づく」）

### 先生向け（一覧 / 詳細 / 設定）
- 基本トーン: **簡潔な常体 + 用語明確化**
- 目標: 判断速度・情報の可読性・作業効率
- 言い回し:
  - 推奨: 「新着3件」「共有シート8件」「コメントを返す」
  - 非推奨: 冗長説明（例: 「こちらを押していただくと…」）
- 語彙:
  - 推奨: 運用用語（例: 「新着」「未読」「共有」「更新」）
  - 非推奨: 感情訴求中心の語（例: 「わくわく」「うれしい」）

## 2. 校正ルール（敬体/常体、語彙）

### 文体ルール
1. 生徒向け本文は敬体（です・ます）で統一する。
2. 先生向けラベルは常体（体言止め含む）を許容し、短くする。
3. 1画面内で敬体/常体を混在させない（例外: 固有名詞・法令文）。

### 語彙ルール
1. 1文は最大45文字を目安にし、1メッセージ1意図に分割する。
2. 主語が曖昧になる代名詞（それ・これ）は連続使用しない。
3. 断定表現（必ず・絶対）は避け、提案表現（〜できます / 〜しましょう）を優先する。
4. 学校運用で誤解のある語は避け、定義済み語彙に寄せる（共有シート・新着・レポート）。

## 3. レビュー観点（チェックリスト）
- 受け手のトーンが合っているか（生徒向け=安心、先生向け=効率）。
- 重要ラベルが短く、一覧で比較しやすいか。
- 日付・時間は固定文言ではなくフォーマッタ経由か。
- 画面文字列が `locales/ja/*.json` などに外出しされているか。
